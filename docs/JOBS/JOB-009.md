**JOB-009: Tích hợp TMDB API - Tự động đồng bộ Đơn lẻ & Hàng loạt**

**1. Context:**
Module Media cần cào dữ liệu từ TMDB. Hệ thống cần hỗ trợ import 1 phim hoặc import tự động hàng loạt hàng trăm phim, đồng thời chống trùng lặp bằng `Name` hoặc `Slug` thay vì dùng `tmdb_id`.

**2. Input:**
- Nguồn dữ liệu ngoài: TMDB API.
- Thành phần cần triển khai: `TmdbSyncService`, `MovieService`, `MoviesController`.
- Endpoint TMDB cần gọi: `/3/genre/movie/list`, `/3/movie/{tmdbId}?append_to_response=credits`, `/3/movie/popular?page={page}`.

**3. Output:**
- **TmdbSyncService (Gọi API TMDB):** Viết hàm `FetchGenresAsync()` gọi `/3/genre/movie/list`; viết hàm `FetchMovieDetailsAsync(int tmdbId)` gọi `/3/movie/{tmdbId}?append_to_response=credits`; viết hàm `FetchPopularMovieIdsAsync(int page)` gọi `/3/movie/popular?page={page}` để lấy danh sách `tmdbId` theo trang.
- **Thuật toán Import (Trong MovieService):** `SyncGenresAsync()` lấy list genres từ TMDB, check `_dbContext.Genres.Any(...)`; nếu chưa có thì add mới. `ImportMovieFromTmdbAsync(int tmdbId)` chạy trong transaction, lấy chi tiết phim và tạo slug, check `_dbContext.Movies.Any(m => m.Slug == slug)`; nếu đã tồn tại thì không throw exception mà trả về `(false, "Đã tồn tại")`; nếu chưa tồn tại thì tạo `Movie` (gán `CreatedByUserId`, URL ảnh), map genres, tạo persons (check trùng theo name), tạo movie credits, commit transaction và trả về `(true, "Thành công")`. `ImportBulkPopularMoviesAsync(int count)` tính số trang cần gọi (ví dụ `count = 100` thì gọi 5 trang, mỗi trang 20 phim), gom toàn bộ `tmdbId`, dùng vòng lặp `foreach` gọi `ImportMovieFromTmdbAsync(id)` cho từng ID; nếu `IsSuccess = false` thì ghi log hoặc tăng biến đếm skipped và tiếp tục; trả về kết quả tổng hợp: "Đã duyệt X phim, Import thành công Y phim, Bỏ qua Z phim trùng lặp".
- **Bổ sung Controller:** Tạo `POST /api/movies/sync-genres`, `POST /api/movies/import/single/{tmdbId}`, `POST /api/movies/import/bulk` (nhận tham số `count`, khuyến nghị tối đa 100-200 phim mỗi lần gọi để tránh timeout HTTP); bổ sung API read-only `GET /api/genres`, `GET /api/persons`.
- **Yêu cầu đầu ra bổ sung:** Báo cáo luồng xử lý Bulk Import, bao gồm cơ chế bỏ qua dữ liệu trùng.

**4. Ràng buộc:**
- Tuyệt đối không sửa file trong thư mục `Entities`.
- Không dùng cột `tmdb_id` để lưu/đối sánh dữ liệu.
- Phải chống trùng lặp bằng `Name` hoặc `Slug`.
