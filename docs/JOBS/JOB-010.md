**JOB-010: Nâng cấp bộ lọc tìm kiếm Phim (Thêm Person Filter)**

**1. Context:**
API `GET /api/movies` hiện tại đã hỗ trợ lọc theo `search`, `genreId` và `status`. Cần bổ sung thêm khả năng lọc phim theo nhân sự (đạo diễn, diễn viên...) để đáp ứng luồng UI khi người dùng click vào hồ sơ một `Person`.

**2. Input:**
- Thành phần cần nâng cấp: `IMovieService`, `MovieService`, `MoviesController`.
- Endpoint cần cập nhật: `GET /api/movies`.
- Điều kiện lọc mới: `Guid? personId = null`.

**3. Output:**
- **Nâng cấp Service Layer (IMovieService & MovieService):** Cập nhật chữ ký hàm `GetMoviesAsync` (hoặc đối tượng query params nếu đang dùng class bọc) để nhận thêm tham số `Guid? personId = null`. Trong implementation của `GetMoviesAsync`, bổ sung mệnh đề LINQ filter vào query: `if (personId.HasValue) query = query.Where(m => m.MovieCredits.Any(mc => mc.PersonId == personId.Value));`.
- **Nâng cấp Controller Layer (MoviesController):** Cập nhật endpoint `GET /api/movies`, bổ sung tham số `[FromQuery] Guid? personId` vào hàm và truyền `personId` xuống cho `MovieService`.
- **Đảm bảo hiệu năng & Testing:** Đảm bảo truy vấn không gây lỗi N+1 query. Với danh sách `MovieDto` phẳng, EF Core phải dịch đoạn `.Any()` thành mệnh đề `EXISTS` trong SQL để tối ưu hiệu năng. Cập nhật unit test (nếu có) để cover trường hợp lọc phim theo `personId`.
- **Yêu cầu đầu ra bổ sung:** Báo cáo các file đã sửa đổi.

**4. Ràng buộc:**
- Tuân thủ kiến trúc Layered hiện tại.
- Không sửa file trong thư mục `Entities`.
