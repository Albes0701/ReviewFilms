**JOB-002: Ánh xạ Database Schema (EF Core PostgreSQL)**

**1. Context (Bối cảnh):**
Chúng ta đã có base project. Bây giờ cần ánh xạ toàn bộ schema từ file SQL thiết kế sang EF Core Entities và cấu hình `ApplicationDbContext` sử dụng PostgreSQL.

**2. Input (Đầu vào):**
- Cấu trúc thư mục hiện tại.
- File thiết kế DB: `DanhGiaPhim.sql` (Tôi cung cấp kèm theo prompt này).

**3. Output (Yêu cầu đầu ra):**
Vui lòng viết code cho các hạng mục sau:
- **Enums:** Tạo các file enum C# trong thư mục `/Enums` tương ứng với các `CREATE TYPE ... AS ENUM` trong SQL (VD: `UserStatus`, `MovieStatus`, `CreditType`,...).
- **Entities:** Tạo các lớp POCO trong thư mục `/Entities` tương ứng với các bảng trong SQL (`User`, `Role`, `Movie`, `Comment`,...). Chú ý:
  - Khai báo đầy đủ Navigation Properties cho các mối quan hệ (1-N, N-N).
  - Sử dụng kiểu `Guid` cho các trường `uuid`.
- **DbContext:** Tạo `ApplicationDbContext.cs` trong thư mục `/Data`.
  - Sử dụng **Fluent API** (ghi đè phương thức `OnModelCreating`) để cấu hình toàn bộ Primary Keys, Foreign Keys, Indexes, và Unique Constraints y hệt như trong SQL.
  - Cấu hình mapping các Enum vào PostgreSQL (sử dụng `.HasPostgresEnum()`).
- **Extensions/DI:** Cập nhật file `/Extensions/ServiceCollectionExtensions.cs` để đăng ký `ApplicationDbContext` với chuỗi kết nối PostgreSQL.

**4. Ràng buộc kỹ thuật:**
- Tuân thủ quy tắc đặt tên: Class và Property dùng `PascalCase` (dù SQL dùng `snake_case`). Hãy để EF Core tự động map tên qua Npgsql hoặc cấu hình rõ ràng trong Fluent API nếu cần.
- Tuyệt đối KHÔNG viết Data Seeder hay Repository/Service ở Job này. Trọng tâm 100% là Entity và DbContext.
- Cung cấp cho tôi lệnh .NET CLI để tạo Migration đầu tiên (VD: `dotnet ef migrations add InitialCreate`).
