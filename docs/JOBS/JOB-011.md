**JOB-012: Nâng cấp Phân quyền - Permission-Based Authorization**

**1. Context:**
Hệ thống hiện tại đang phân quyền theo Role (`ClaimTypes.Role`) thông qua `[Authorize(Roles = "ADMIN")]`. Database đã có sẵn cấu trúc `Role -> RolePermission -> Permission`. Cần nâng cấp mã nguồn để xác thực quyền truy cập dựa trên mã Permission (ví dụ: `movies:import`) nhằm phân quyền linh hoạt hơn theo chuẩn Enterprise (`Policy-Based Authorization`).

**2. Input:**
- Thành phần nghiệp vụ liên quan: `AuthService`, `JwtTokenGenerator`, tầng Security Authorization, các API `Movies`.
- Cấu trúc dữ liệu quyền sẵn có: `UserRoles -> Role -> RolePermissions -> Permission`.
- Quyền mục tiêu cần map theo endpoint: `genres:sync`, `movies:import`, `movies:create`, `movies:update`, `movies:delete`.

**3. Output:**
- **Cập nhật truy vấn dữ liệu & DTOs (Tầng Service):** Trong `AuthService.cs`, cập nhật hàm `GetUserWithRolesAsync`; khi query `User`, dùng `.Include` và `.ThenInclude` đi sâu qua `UserRoles -> Role -> RolePermissions -> Permission` để lấy toàn bộ permission code user sở hữu. Cập nhật `UserProfileDto` và `AuthResponse` bổ sung mảng `Permissions` (kiểu `string[]`). Ở phần map DTO trong `AuthService.cs`, gom tất cả `permission.code`, loại bỏ trùng lặp (`Distinct`) và gán vào `Permissions`.
- **Đưa Permission vào JWT Token (`JwtTokenGenerator.cs`):** Cập nhật hàm sinh token; ngoài claim Role, duyệt mảng permission code và thêm vào danh sách claims của JWT dưới dạng custom claim (ví dụ: `ClaimTypes.AuthorizationDecision` hoặc `"permissions"`).
- **Xây dựng Custom Authorization Policy (Tầng Security):** Tạo custom attribute `[HasPermission(string permission)]` kế thừa từ `AuthorizeAttribute`. Tạo `PermissionRequirement` kế thừa `IAuthorizationRequirement`. Tạo `PermissionAuthorizationHandler` kế thừa `AuthorizationHandler<PermissionRequirement>`; trong `HandleRequirementAsync`, đọc claim `"permissions"` từ JWT của user, nếu chứa mã quyền yêu cầu thì `context.Succeed(requirement)`. Đăng ký handler và cấu hình authorization policy provider trong `AuthModuleExtensions.cs`.
- **Cập nhật lại các Controller (Tầng API):** Quét toàn bộ API hiện tại, thay các `[Authorize(Roles = "ADMIN")]` bằng `[HasPermission("...")]` theo mapping: `POST /api/movies/sync-genres -> [HasPermission("genres:sync")]`, `POST /api/movies/import/* -> [HasPermission("movies:import")]`, `POST /api/movies -> [HasPermission("movies:create")]`, `PUT /api/movies/{id} -> [HasPermission("movies:update")]`, `DELETE /api/movies/{id} -> [HasPermission("movies:delete")]`.
- **Yêu cầu đầu ra bổ sung:** Báo cáo chi tiết các class policy/handler đã tạo, cách cấu hình DI và kết quả test thử một endpoint bị chặn quyền.
