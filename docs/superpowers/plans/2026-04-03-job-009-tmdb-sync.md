# JOB-009 TMDB Sync Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add TMDB integration for genre sync, single-movie import, bulk popular import, and read-only genre/person endpoints without using `tmdb_id` for persistence or deduplication.

**Architecture:** Use `ITmdbSyncService`/`TmdbSyncService` for external API calls and TMDB response mapping, while `IMovieService`/`MovieService` own deduplication, transactions, entity creation, and summary reporting. Controllers stay thin and return `ApiResponse<T>` for every new endpoint.

**Tech Stack:** ASP.NET Core Web API, EF Core, MySQL entities, `HttpClient`, `IOptions`, xUnit, InMemory EF tests
